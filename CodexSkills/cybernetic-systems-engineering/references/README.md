# References

## 使用顺序

1. 先读 `SKILL.md`
   - 获取触发条件、执行骨架、控制合同、验证分层
2. 需要更完整理论与设计视角时，再读：
   - `gda-framework.md`

## 文件说明

### `gda-framework.md`

包含：

- 五大维度：
  - 第一性原理
  - 公理化思维
  - 多模型思维
  - 类比迁移
  - 反馈与验证
- GDA 四步法：
  - Axiom & Boundary
  - Multi-model Construction
  - Cybernetic Control
  - Closed-loop Observability
- 面向资深 SWE / 架构师的架构与测试设计启示

## 读取建议

只有在以下场景才需要读取 `gda-framework.md`：

- 任务复杂、跨模块、跨语言、跨环境
- 需要做系统级审计或总体设计
- 需要给出测试分层、门禁、handoff、长期演进建议
- 需要解释“为什么要这样设计”，而不是只给 patch
