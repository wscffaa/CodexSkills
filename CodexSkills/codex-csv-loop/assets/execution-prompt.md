你将基于当前任务注入的 CSV ledger 作为任务驱动台账；如果没有显式注入，则回退到本地 `docs/workflow/ledgers/issues.csv`。按“读取 → 开发 → 验证 → 回卷 → 更新状态”的循环推进工作。

请严格遵守以下规则：

1. 任务读取与锁定
- 读取注入的 CSV ledger 路径；若未提供，则读取 `docs/workflow/ledgers/issues.csv`，选择首个可执行原子任务。
- 在开始写代码前，先把该行状态锁定为 `进行中`。
- 一次只处理一个原子任务。

2. 开发与验证
- 仅根据当前行的 `Content`、`Acceptance Criteria`、`Review Requirements` 工作。
- 验收必须基于真实命令、测试、日志、产物或可观察行为。
- 如果验收标准不明确，不要假装完成，先报告缺口。

3. Review + Rewind
- 对照 `Review Requirements` 检查回归、边界条件和残留修改。
- 确认当前任务影响边界内没有未清理的半成品。

4. 状态更新
- 只有在证据充分时才把当前行更新为 `已完成`。
- 然后继续读取下一条可执行任务。
- 若无剩余任务，输出总结并停止。

5. 遇到阻塞
- 若需要权限、环境、密钥或关键上下文，明确指出阻塞点。
- 若中断执行，记录当前任务、阻塞原因和建议续跑入口。
